# 24時間シート（サイドバー）仕様書

## 📋 概要
24時間シート（ケアプラン）は、入居者様の1日の生活スケジュールとケア内容を時系列で管理する中核機能です。左側のサイドバーに表示され、職員の行動指針となります。

---

## 🗃️ データベース構造

### care_plans テーブル

| カラム名 | 型 | 説明 | 必須 | デフォルト |
|---------|-----|------|------|-----------|
| id | INTEGER | 主キー | ✅ | AUTO |
| resident_id | INTEGER | 入居者ID | ✅ | - |
| time | TEXT | 時刻（例：07:00、08:00） | ✅ | - |
| activity | TEXT | 活動名（例：起床・洗面、朝食） | ✅ | - |
| details | TEXT | 詳細な介助方法・留意点 | ❌ | NULL |
| status | TEXT | ステータス：`plan`（計画）/ `fit`（フィット済み） | ✅ | 'plan' |
| display_order | INTEGER | 表示順序（昇順） | ✅ | 0 |
| created_at | DATETIME | 作成日時 | ✅ | CURRENT_TIMESTAMP |
| updated_at | DATETIME | 更新日時 | ✅ | CURRENT_TIMESTAMP |

**外部キー制約:**
- `resident_id` → `residents(id)` ON DELETE CASCADE

**インデックス:**
- `idx_care_plans_resident_id` ON `resident_id`

---

## 🎯 ステータスの意味

### `plan`（計画段階）
- **意味**: 暫定的なケアプラン。入所初期や調整中の項目。
- **表示**: 白背景、グレーの時刻バッジ
- **用途**: Day1〜Day7の試行期間中の計画

### `fit`（フィット済み）
- **意味**: 入居者様に合っていることが確認されたケア内容。
- **表示**: 水色背景、ティール色の時刻バッジ、チェックマークアイコン
- **用途**: Day8以降で確定したケア内容

## 🔌 API仕様

### 1. 24時間シート取得
**エンドポイント:** `GET /api/residents/:id/care-plans`

**レスポンス例:**
```json
[
  {
    "id": 1,
    "resident_id": 1,
    "time": "07:00",
    "activity": "起床・洗面",
    "details": "カーテンを開け日光を入れる。右側から声をかける。",
    "status": "fit",
    "display_order": 1,
    "created_at": "2026-03-19 10:00:00",
    "updated_at": "2026-03-21 08:00:00"
  },
  {
    "id": 2,
    "resident_id": 1,
    "time": "08:00",
    "activity": "朝食",
    "details": "椅子に深く座るよう促す。お茶は熱めを希望。",
    "status": "plan",
    "display_order": 2,
    "created_at": "2026-03-19 10:00:00",
    "updated_at": "2026-03-19 10:00:00"
  }
]
```

**ソート順:**
- `display_order ASC`（表示順序の昇順）

---

### 2. 24時間シート項目更新
**エンドポイント:** `PUT /api/care-plans/:id`

**リクエストボディ:**
```json
{
  "time": "08:15",
  "activity": "朝食",
  "details": "椅子に深く座るよう促す。コーヒー（ブラック、砂糖1）を提供。",
  "status": "fit"
}
```

**更新可能項目:**
- `time`: 時刻
- `activity`: 活動名
- `details`: 詳細内容
- `status`: ステータス（`plan` / `fit`）

**レスポンス:**
```json
{
  "success": true
}
```

**注意:**
- `updated_at`は自動更新される
- `display_order`や`resident_id`は更新不可

## 📊 データフロー

### 1. 初回読み込み
```
ページロード
   ↓
GET /api/residents/1/care-plans
   ↓
display_order順にソート済みデータ取得
   ↓
State (carePlans) に格納
   ↓
UI描画（時系列で表示）
```

### 2. 付箋との連携
```
24時間シート表示
   ↓
各項目のtime（例：08:00）を取得
   ↓
stickyNotesをtimeでグループ化
   ↓
該当時刻に紐づく付箋を表示
```

### 3. ステータス更新フロー（将来実装）
```
付箋「とりいれる」をクリック
   ↓
付箋のstatusを'adopted'に更新
   ↓
【今後】付箋の内容を24時間シートに反映
   ↓
PUT /api/care-plans/:id
   ↓
detailsを更新 or statusを'fit'に変更
   ↓
UI再描画
```

## 🎯 現在の実装状況

### ✅ 実装済み
1. **表示機能**
   - 時系列での項目表示
   - ステータス別の視覚的区別（plan / fit）
   - タイムライン表示
   - スクロール対応

2. **データ取得**
   - 入居者IDベースの取得
   - display_order順のソート

3. **付箋連携**
   - 時刻ベースで付箋を表示
   - 付箋のアクションボタン

### ⚠️ 未実装・制約

1. **編集機能なし**
   - 現在はデータベース直接編集のみ
   - UI上での時刻・活動名・詳細の編集不可
   - 項目の追加・削除不可

2. **並び替え機能なし**
   - `display_order`は固定
   - ドラッグ&ドロップでの並び替え不可

3. **自動更新なし**
   - 付箋採用時に24時間シートは自動更新されない
   - 手動でDBまたはAPIを叩く必要あり

4. **履歴管理なし**
   - 変更履歴の記録なし
   - 誰がいつ変更したかの追跡不可

5. **バージョン管理なし**
   - 複数バージョンの管理不可
   - Day1版、Day14版などの保存・比較不可

6. **印刷・エクスポート機能なし**
   - PDF出力不可
   - Excel/CSV出力不可

## 📊 データ整合性とバリデーション

### 必須バリデーション
1. **時刻形式**
   - HH:MM形式（例：07:00、23:59）
   - 24時間表記
   - 00:00〜23:59の範囲

2. **活動名**
   - 最大50文字
   - 空白不可

3. **display_order**
   - 重複不可
   - 連番推奨（1, 2, 3, ...）

4. **status**
   - `plan` または `fit` のみ許可

### 推奨事項
- 時刻は1時間単位または30分単位で設定
- 1日あたり6〜15項目が適切
- 詳細は200文字以内が望ましい
